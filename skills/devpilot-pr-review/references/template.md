# Review Template

Render the skeleton below into the posted review. Fill every placeholder; do not leave `<...>` text in the output.

## Skeleton

```
<!-- devpilot-pr-review (devpilot <version>) -->
Hi @<author-handle>,

Thanks for the change. Review below.

## PR Review: <title / #number>

### TL;DR
<1–2 sentences: safe to merge? single most important thing to address?>

### Unknown-Unknowns Sweep
1. Local pattern fit: <finding or "matches convention in X">
2. Blast radius: <finding>
3. Known pitfalls (incl. security/data/reversibility): <finding>
4. Stale-training check: <finding or "N/A">
5. Hand-rolled vs. off-the-shelf: <finding or "N/A">

### Behavior Findings
#### [Blocking | Should-fix | Consider | Nit] <short title>
- **Where:** `path/file.go:42`
- **Behavior today on this branch:** <what the code actually does>
- **Why that's a problem:** <impact on users / data / operability>
- **Suggested change:** <concrete direction>
- **Confidence:** high | medium | low — <one-line reason if not high>

### What's working well
<2–3 bullets worth preserving>

### Nits (optional)
<style / naming / wording, one line each>

### Open Questions (optional — omit if empty)
<only things the code could not answer, one line each>

---

> **Note:** This review was generated automatically by Claude Code. It is not a substitute for human judgment. Treat every finding — including severity labels — as a prompt for the author and human reviewers to verify, not as a final verdict. Approvals do not waive human review.

<!--
Generated by devpilot-pr-review (devpilot <version>, e.g. v0.12.2)
Model: <model id, e.g. claude-opus-4-7>
Review mode: <request-changes | comment | approve>
-->
```

## Field rules

- **`<version>`** — resolve via `devpilot --version` at post time. Take the `vX.Y.Z` token. Fall back to `unknown` if the command is unavailable. The same value goes in both the leading marker and the trailing HTML comment.
- **`<author-handle>`** — resolve via `gh pr view --json author -q .author.login`. Fall back to "Hi there," (no handle) if unavailable.
- **Language** — render every section (TL;DR, findings, disclaimer, Open Questions, metadata) in the PR's language. The disclaimer must be translated while preserving its meaning: automated, not authoritative, human judgment required.
- **Severity labels** — describe impact-if-true. See the severity rubric in SKILL.md.
- **Confidence labels** — describe how sure you are. `high`, `medium`, or `low`, independent of severity. Low confidence never demotes severity automatically.
- **Empty sections** — `Nits` and `Open Questions` are optional; omit the heading entirely if empty. TL;DR, Unknown-Unknowns Sweep, Behavior Findings, What's working well, the disclaimer, and both metadata blocks are always present.
- **Post mode** — derived from the highest-severity finding:
  - Any Blocking → `--request-changes`
  - Only Should-fix / Consider / Nit → `--comment`
  - Zero findings → `--approve`

For a fully-filled reference, see `example-review.md`.
